# Phase 04 — Concept Development: BRIEF

## Purpose

Generate, explore, and select product concepts that meet the definition from Phase 03. This phase answers: **"How might this product work, look, and feel — and which direction is best?"**

This is the most creative phase. The goal is to explore broadly before narrowing. Multiple concepts should be generated and evaluated against the requirements and criteria established in Phase 03.

---

## Inputs Needed

- Phase 03 HANDOVER.md — product definition, requirements, design principles, evaluation criteria
- Phase 02 research findings for reference (competitor designs, material options, manufacturing methods)
- Any existing design direction, mood boards, or visual references

---

## Sub-Task Menu

### Core (recommended for all projects)

- [ ] **Concept ideation** — Generate multiple distinct concepts (aim for 3-5 minimum). Each should represent a meaningfully different approach, not just cosmetic variations.
- [ ] **Concept description & visualisation** — Describe each concept clearly. Include rough sketches, AI-generated visuals, reference images, or mood boards. Someone who wasn't in the room should be able to understand each concept.
- [ ] **Concept evaluation matrix** — Score each concept against the evaluation criteria from Phase 03. Use a structured matrix to make the comparison explicit and fair.
- [ ] **Concept selection & rationale** — Choose the winning concept(s) and document why. Record what was sacrificed in the alternatives.

### Conditional (select based on product type)

- [ ] **Form factor exploration** — For physical products where shape/size is a key decision: explore different form factors, ergonomics, proportions.
- [ ] **Mechanical concept exploration** — How do moving parts, mechanisms, or assemblies work? Explore different mechanical approaches.
- [ ] **Graphic & visual design concepts** — For products where graphic design is central (games, packaging-heavy products): explore different visual styles, illustration approaches, typography.
- [ ] **Interaction design concepts** — How does the user interact with the product? Different control schemes, feedback mechanisms, user flows.
- [ ] **Material & finish concept exploration** — Different material and finish combinations: how they look, feel, and affect cost/manufacturing.
- [ ] **Packaging concept development** — Packaging as part of the product experience: structural concepts, graphic approaches, unboxing flow.
- [ ] **Component architecture** — For products with multiple parts: how components relate, assemble, and interact.

### Optional / Deep Dive

- [ ] **Quick feasibility check** — Before fully committing: get a sanity check on the preferred concept's manufacturability, cost, and technical risk.
- [ ] **User concept testing** — Show concept descriptions/visuals to potential users for early feedback. Can be informal.
- [ ] **Hybrid concept development** — Combine the best elements from multiple concepts into an optimised hybrid.
- [ ] **Risk-per-concept assessment** — For each concept, what's the biggest risk? Which has the best risk profile overall?

---

## Done Criteria

1. At least 3 meaningfully different concepts were generated
2. Each concept is described and visualised clearly enough to evaluate
3. Concepts were evaluated against defined criteria in a structured way
4. A primary concept is selected with clear rationale
5. The HANDOVER captures the selected concept and key design parameters for Phase 05

---

## Notes

- Don't fall in love with the first idea. The discipline of generating alternatives almost always reveals a better direction or strengthens the original through comparison.
- For products entering with existing design direction (e.g., gameplay/artwork already done), concept development focuses on the un-defined aspects: physical form, packaging, component specification, manufacturing approach.
- AI image generation can be valuable here for quick concept visualisation, but be explicit about what's a rough direction vs. a target design.
- The concepts don't need to be fully resolved — that's Phase 05's job. They need to be clear enough to choose between.
